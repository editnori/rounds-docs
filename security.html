Security review

1. Purpose and scope

Rounds is a side-panel companion that turns the Memo to Chart alerts already on the clinician's screen into a triage inbox, and uses CareLogic's own links and forms when the clinician acts. CareLogic remains the sole system of record; Rounds stores no clinical record of its own. The walkthrough shows every claim in this section live, on fictional sample data; ask for the 30-minute demo and verify as you read.

Explicit negatives, up front:

  • No servers. The extension has no backend, no account system, and no sync.
  • No network access. The manifest's content security policy is default-src 'none': the browser refuses any request the extension might try to make. v0.2 removed the last external fetch (a font) by bundling it.
  • No telemetry or analytics. Nothing is measured off the machine.
  • No AI. Memo grouping is deterministic word matching; the complete vocabulary is printed inside the panel. There is no model, no learning, no cloud inference.
  • No patient-facing surface. Rounds is visible only to the logged-in clinician.

2. Architecture

Three components, roughly 2,500 lines of plain JavaScript plus one classifier file. Zero third-party dependencies, no build step. The code that ships is the code that runs, reviewable in one sitting. We offer a live walkthrough of every file.

The absence list is the stronger half: no tabs, no scripting, no webRequest, no cookies, no downloads, no nativeMessaging, no externally_connectable, no web_accessible_resources. Rounds cannot enumerate tabs, inject code on demand, observe traffic, read cookies, or be messaged by any web page.

7. Threats

  • Injected/scraped content (XSS). Everything read from the page is HTML-escaped before rendering, and the CSP blocks script injection into the panel. We ship a regression memo containing an <img onerror> payload in the staging seed to prove the sink is closed.
  • Wrong-patient action. Commands address rows by parsed identity (chart ID + sent time), re-resolved against the live table at click time. Zero or multiple matches: the command refuses rather than clicking the wrong row, and the refusal is written to the visible action log. The memo-form write carries the same protection: it refuses unless the page holding the form displays the target client's chart ID ("refused (different client on screen)" in the action log). A refusal is the guard working; a misdirected click is a stop-ship defect.
  • Session misuse. All commands originate from a clinician gesture in the panel. Web pages cannot message the extension (externally_connectable absent). Rounds never signs and never submits a form.
  • Supply chain. No third-party code, no package manager, no build step. The delivered zip is hashed; the hash is in the delivery note.
  • Shared workstations. Rounds inherits the agency's lock/timeout policy; the 30-minute live-data expiry and 60-day triage prune bound what an unlocked session exposes. Q3/Q7 below.
  • Host upgrades. If a CareLogic update changes the page markup, scanning fails safe: the panel shows nothing new and performs no writes; selectors live in one tunable block at the top of the content script.

8. Auto-grouping

Grouping is a fixed, versioned set of word rules (panel/rules.js, rule set v1) evaluated locally against each memo's narrative. Identical input produces identical grouping on every machine. The panel prints every rule, every term, a live match count, and a switch per rule plus a master switch. Clinician overrides always win: a manually removed group is recorded per-memo and never re-applied by rules; a manually added group is untouchable by rules. There is no learning component; behavior changes only through a shipped, reviewable update.

9. Pilot success criteria

  1. Median memo triage ≤ 30 seconds, against the ~6-click / 2-page-load baseline (observed, not instrumented).
  2. Zero wrong-row actions across the pilot. Logged refusals are acceptable and expected; any misdirected click stops the pilot.
  3. ≤ 1 selector patch needed after week 1 against the agency's CareLogic build.
  4. ≥ 4 of 5 pilot clinicians still using the panel in week 4, unprompted.
  5. IT verification passes at close: zero extension-origin network requests; storage contents match §4.
